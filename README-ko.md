<summary><h2>✨ 기능 목록</h2></summary>

### 1. 호출 간격 제한
매번 LLM API 호출 전에 강제로 쿨다운 시간을 대기시켜, 짧은 시간에 대량의 요청으로 API 잔액이 소모되는 것을 방지합니다.

> 예: 3초 간격으로 설정하면, 봇이 메시지에 답장한 후 최소 3초를 기다려야 다음 API를 호출할 수 있습니다.

### 2. 호출 횟수 제한 + 쿨다운 리셋
봇이 지정한 연속 메시지 수에 도달하면 API 호출을 자동으로 일시 정지하고, 일정 시간 후 자동으로 복구합니다.

> 예: 최대 호출 10회, 쿨다운 5분으로 설정. 봇이 10개의 메시지를 보낸 후 5분간 "조용해지고", 그 후 자동으로 작동을 재개합니다.

### 3. 조용한 시간 (정시 API 차단)
지정한 시간대에 API 호출을 완전히 금지하여, 봇이 심야에 그룹 채팅에서 메시지를 보내 방해하는 것을 막습니다.

> 예: 조용한 시간을 23:00~07:00으로 설정. 이 시간대에는 봇이 API를 호출하지 않고, LLM이 필요한 메시지에 답장하지 않습니다.

### 4. 일일 호출 쿼터
하루에 허용되는 API 호출 총 횟수로, 다음 날 자동으로 리셋됩니다. 횟수 제한과 달리 일일 쿼터는 자정을 기준으로 계산되는 하드 리미트입니다.

> 예: 일일 쿼터를 100회로 설정. 당일 누적 호출이 100회에 도달하면 모든 요청이 차단되고, 다음 날 0시에 자동 복구됩니다.

### 5. 화이트리스트 / 블랙리스트
- **화이트리스트**: 제한의 영향을 받지 않는 사용자 ID를 지정. 관리자는 제한 없음
- **블랙리스트**: 지정한 사용자를 즉시 거부. 쿼터를 소모하지 않음

> 예: 화이트리스트를 `123456,789012`로 설정. 이 두 사용자의 모든 요청은 바로 통과합니다.

### 6. 커스텀 거부 메시지
요청이 차단되었을 때 사용자에게 자동으로 답장하는 메시지. 비워두면 조용히 차단합니다.

> 예: 거부 메시지를 "요청이 너무 많습니다. 잠시 후 다시 시도해 주세요~"로 설정. 차단된 사용자에게 이 메시지가 표시됩니다.

### 7. 호출 통계 패널
통계 데이터를 확인하는 두 가지 방법:

- **채팅 통계**: `/webui`를 전송하여 채팅 내에서 텍스트 기반 통계를 확인. 하단에 WebUI 패널을 여는 안내 포함
- **WebUI 패널**: 관리자가 `开启统计面板`을 전송하여 브라우저 기반 통계 페이지를 실행. LAN 접속 + Token 인증 지원

> WebUI 패널은 브라우저에서 오늘의 호출 횟수, 누적 허용/차단, 허용률, 현재 설정 상태 등을 표시하며, 원클릭 새로고침을 지원합니다.

### 8. 대화 차단
개별 사용자 또는 전체 사용자가 짧은 시간에 API 잔액을 소진하는 것을 방지. 두 가지 모드 (둘 중 하나 선택):

- **사용자별 모드**: 각 사용자를 독립적으로 카운트. 상한에 도달해도 해당 사용자만 차단. 다른 사용자에게는 영향 없음
- **전체 모드**: 모든 사용자를 합산하여 카운트. 상한에 도달하면 모두가 차단됨

> 개인 채팅과 그룹 채팅은 별도로 계산되며, 각 그룹/개인 채팅에서 독립적으로 카운트. 다음 날 자동 리셋.
>
> 예: 사용자별 모드, 상한 20회, 쿨다운 30분. 사용자 A가 20회 연속 대화하면 API가 30분간 차단되지만, 그 동안 사용자 B는 영향을 받지 않습니다.

### 9. 시간대별 속도 제한
시간별로 다른 속도 제한 파라미터(간격, 횟수, 쿨다운)를 설정. 낮에는 느슨하게, 밤에는 엄격하게.

### 10. 쿼터 초과 알림
일일 쿼터 사용량이 80%에 도달하면 관리자에게 자동으로 알림.

### 11. 그룹별 쿼터
그룹마다 다른 일일 쿼터를 설정. 서로 영향을 주지 않음.

### 12. 관리자 명령어
- `重置限频` — 모든 카운터와 쿨다운 리셋
- `重置冷却` — 쿨다운 상태만 리셋
- `重置配额` — 일일 쿼터만 리셋
- `重置对话` — 대화 차단 카운터만 리셋
- `清空日志` — 차단 로그 비우기
- `导出日志` — 차단 로그를 텍스트로 내보내기

<summary><h2>📦 설치</h2></summary>

**방법 1: GitHub에서 클론 (권장)**

```bash
cd AstrBot/data/plugins/
git clone https://github.com/xiaohondan/astrbot_plugin_api_limiter.git
```

**방법 2: 플러그인 마켓에서 설치**

AstrBot WebUI → 플러그인 마켓에서 **API 호출 제한기** 또는 **小红蛋**을 검색하고, 설치를 클릭.

**방법 3: 수동 설치**

1. 이 저장소의 ZIP 아카이브를 다운로드
2. AstrBot의 `data/plugins/` 디렉토리에 압축 해제
3. AstrBot 재시작

<summary><h2>⚙️ 설정 방법</h2></summary>

설치 후 AstrBot WebUI → 플러그인 관리 → **API 호출 제한기** → 설정에서 설정합니다:

| 설정 항목 | 타입 | 기본값 | 설명 |
|:---|:---:|:---:|:---|
| **호출 쿨다운 간격** | 정수 | 3 | 각 API 호출 전 대기할 초. 0이면 제한 없음 |
| **최대 호출 횟수** | 정수 | 0 | 연속 메시지 수 도달 시 API 일시 정지. 0이면 제한 없음 |
| **쿨다운 대기 시간** | 정수 | 0 | 상한 도달 후 대기할 분. 0이면 자동 복구 없음 |
| **일일 호출 쿼터** | 정수 | 0 | 하루 최대 API 호출 횟수, 다음 날 자동 리셋. 0이면 제한 없음 |
| **조용한 시간 시작** | 문자열 | 빈칸 | 형식 `HH:MM` (예 `23:00`). 비워두면 비활성화 |
| **조용한 시간 종료** | 문자열 | 빈칸 | 형식 `HH:MM` (예 `07:00`). 비워두면 비활성화 |
| **화이트리스트 사용자 ID** | 문자열 | 빈칸 | 쉼표로 구분. 비워두면 화이트리스트 없음 |
| **블랙리스트 사용자 ID** | 문자열 | 빈칸 | 쉼표로 구분. 비워두면 블랙리스트 없음 |
| **시간대별 속도 제한** | 문자열 | 빈칸 | JSON 형식, 시간별 속도 제한 파라미터 설정 |
| **그룹별 쿼터** | 문자열 | 빈칸 | 형식: 그룹ID:쿼터, 세미콜론으로 구분 |
| **커스텀 거부 메시지** | 문자열 | 빈칸 | 차단 시 답장할 메시지. 비워두면 조용히 차단 |
| **통계 패널 포트** | 정수 | 6285 | WebUI 통계 패널의 HTTP 포트 |
| **통계 패널 토큰** | 문자열 | 빈칸 | 패널 접속에 필요한 Token. 비워두면 인증 없음 |
| **대화 차단 모드** | 드롭다운 | 빈칸 | `사용자별` 또는 `전체`. 비워두면 비활성화 |
| **대화 차단 상한** | 정수 | 0 | 이 값에 도달하면 API 차단 |
| **대화 차단 쿨다운** | 정수 | 0 | 상한 도달 후 복구까지의 분 |

### 설정 예시

**시나리오 1: API 잔액의 빠른 소모 방지**
```
호출 쿨다운 간격: 3초
최대 호출 횟수: 20회
쿨다운 대기 시간: 10분
일일 호출 쿼터: 100회
커스텀 거부 메시지: 요청이 너무 많습니다. 잠시 후 다시 시도해 주세요~
```
> 봇은 3초마다 최대 1건 답장, 20건 연속 전송 후 10분 휴식, 하루 최대 100회.

**시나리오 2: 심야 방해 방지**
```
조용한 시간 시작: 23:00
조용한 시간 종료: 07:00
커스텀 거부 메시지: 지금은 조용한 시간입니다. 내일 다시 와주세요~
```
> 밤 11시부터 아침 7시까지 봇이 API를 완전히 호출하지 않습니다.

**시나리오 3: 전면 보호 + 관리자 화이트리스트**
```
호출 쿨다운 간격: 3초
최대 호출 횟수: 15회
쿨다운 대기 시간: 5분
일일 호출 쿼터: 200회
조용한 시간 시작: 00:00
조용한 시간 종료: 07:00
화이트리스트 사용자 ID: 123456789
```
> 관리자는 제한 없음. 다른 사용자는 낮에 3초마다 1건 답장, 심야에는 완전 차단.

<summary><h2>📊 통계 패널</h2></summary>

**채팅 통계**: `/webui`를 전송하여 API 속도 제한 통계 데이터를 확인합니다.

**WebUI 패널**: 관리자는 다음 명령어로 웹 패널을 제어합니다:

```bash
开启统计面板    # 시작, 접속 URL(Token 포함) 반환
关闭统计面板    # 패널 종료
```

브라우저에 표시되는 내용:
- 오늘의 호출 횟수 / 일일 쿼터
- 누적 허용 / 누적 차단 / 허용률
- 쿨다운 상태 프로그레스 바
- 현재 모든 설정 상태
- 최근 차단 로그

LAN 접속 + Token 인증을 지원하여 승인되지 않은 조회를 방지합니다.

<summary><h2>🔧 호환성</h2></summary>

- **AstrBot 버전**: >= 4.16, < 5
- **지원 플랫폼**: AstrBot이 지원하는 모든 플랫폼 (QQ, Telegram, Discord 등)
