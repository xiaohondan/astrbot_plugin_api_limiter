<summary><h2>✨ 機能一覧</h2></summary>

### 1. 呼び出し間隔制限
各 LLM API 呼び出しの前に強制的にクールダウン時間を設け、短時間に大量のリクエストで API 残高が消費されるのを防ぎます。

> 例：3秒間隔を設定すると、ボットはメッセージに返信した後、少なくとも3秒待ってから次の API を呼び出します。

### 2. 回数制限 + クールダウンリセット
ボットが指定した連続メッセージ数に達すると、自動的に API 呼び出しを一時停止し、一定時間後に自動的に復帰します。

> 例：最大呼び出し回数10回、クールダウン5分を設定。ボットは10件のメッセージを送信した後、5分間「静かに」なり、その後自動的に動作を再開します。

### 3. 静穏時間（定時API遮断）
指定した時間帯に API 呼び出しを完全に禁止し、深夜にボットがグループチャットでメッセージを送信して迷惑をかけるのを防ぎます。

> 例：静穏時間を23:00〜07:00に設定。この時間帯はボットが API を呼び出さず、LLM が必要なメッセージには返信しません。

### 4. 1日の呼び出しクオータ
1日に許可される API 呼び出しの合計回数で、翌日に自動的にリセットされます。回数制限とは異なり、1日のクオータは自然日で計算されるハードリミットです。

> 例：1日のクオータを100回に設定。当日の累計呼び出しが100回に達すると全てのリクエストがブロックされ、翌日0時に自動復帰します。

### 5. ホワイトリスト / ブラックリスト
- **ホワイトリスト**：制限の影響を受けないユーザーIDを指定。管理者は一切制限されません
- **ブラックリスト**：指定したユーザーを直接拒否。クオータを消費しません

> 例：ホワイトリストを `123456,789012` に設定。これらのユーザーの全リクエストは直接通過します。

### 6. カスタム拒否メッセージ
リクエストがブロックされたときにユーザーへ自動返信するメッセージ。空欄の場合はサイレントブロックします。

> 例：拒否メッセージを「リクエストが多すぎます。しばらく待ってください〜」に設定。ブロックされたユーザーにはこのメッセージが表示されます。

### 7. 呼び出し統計パネル
統計データを確認する2つの方法：

- **チャット統計**：`/webui` を送信してチャット内でテキストベースの統計を表示。下部に WebUI パネルを開くための案内付き
- **WebUI パネル**：管理者が `开启统计面板` を送信してブラウザベースの統計ページを起動。LAN アクセス + Token 認証に対応

> WebUI パネルはブラウザで本日の呼び出し回数、累計許可/ブロック数、許可率、現在の設定状態などを表示し、ワンクリックリロードに対応。

### 8. 会話遮断
個別ユーザーまたは全ユーザーによる短時間での API 残高枯渇を防止。2つのモード（いずれか1つ）：

- **ユーザー別モード**：各ユーザーを独立してカウント。上限に達してもそのユーザーのみブロック。他のユーザーには影響なし
- **グローバルモード**：全ユーザーの合計でカウント。上限に達すると全員がブロック

> 個チャットとグループチャットは別々に計算され、各グループ/個チャットで独立してカウント。翌日に自動リセット。
>
> 例：ユーザー別モード、上限20回、クールダウン30分。ユーザーAが20回連続会話すると API が30分間遮断されるが、その間ユーザーBは影響を受けません。

### 9. 時間帯別レート制限
時間ごとに異なるレート制限パラメータ（間隔、回数、クールダウン）を設定。昼間は緩く、夜間は厳しく。

### 10. クオータ警告
1日のクオータが80%に達すると管理者に自動で通知。

### 11. グループ別クオータ
グループごとに異なる1日のクオータを設定。互いに影響しません。

### 12. 管理者コマンド
- `重置限频` — 全てのカウンターとクールダウンをリセット
- `重置冷却` — クールダウン状態のみリセット
- `重置配额` — 1日のクオータのみリセット
- `重置对话` — 会話遮断カウンターのみリセット
- `清空日志` — ブロックログをクリア
- `导出日志` — ブロックログをテキストとしてエクスポート

<summary><h2>📦 インストール</h2></summary>

**方法1：GitHubからクローン（推奨）**

```bash
cd AstrBot/data/plugins/
git clone https://github.com/xiaohondan/astrbot_plugin_api_limiter.git
```

**方法2：プラグインマーケットからインストール**

AstrBot WebUI → プラグインマーケット で **API呼出制限器** または **小红蛋** を検索し、インストールをクリック。

**方法3：手動インストール**

1. このリポジトリの ZIP アーカイブをダウンロード
2. AstrBot の `data/plugins/` ディレクトリに解凍
3. AstrBot を再起動

</details>

<summary><h2>⚙️ 設定方法</h2></summary>

インストール後、AstrBot WebUI → プラグイン管理 → **API呼出制限器** → 設定 で設定します：

| 設定項目 | タイプ | デフォルト | 説明 |
|:---|:---:|:---:|:---|
| **呼び出しクールダウン間隔** | 整数 | 3 | 各 API 呼び出し前に待機する秒数。0で無制限 |
| **最大呼び出し回数** | 整数 | 0 | 連続メッセージ数で API を一時停止。0で無制限 |
| **クールダウン待機時間** | 整数 | 0 | 上限到達後に待機する分数。0で自動復帰なし |
| **1日の呼び出しクオータ** | 整数 | 0 | 1日の最大 API 呼び出し回数、翌日自動リセット。0で無制限 |
| **静穏時間開始** | 文字列 | 空 | 形式 `HH:MM`（例 `23:00`）。空欄で無効 |
| **静穏時間終了** | 文字列 | 空 | 形式 `HH:MM`（例 `07:00`）。空欄で無効 |
| **ホワイトリストユーザーID** | 文字列 | 空 | カンマ区切り。空欄でホワイトリストなし |
| **ブラックリストユーザーID** | 文字列 | 空 | カンマ区切り。空欄でブラックリストなし |
| **時間帯別レート制限** | 文字列 | 空 | JSON形式、時間ごとにレート制限パラメータを設定 |
| **グループ別クオータ** | 文字列 | 空 | 形式：グループID:クオータ、セミコロン区切り |
| **カスタム拒否メッセージ** | 文字列 | 空 | ブロック時に返信するメッセージ。空欄でサイレントブロック |
| **統計パネルポート** | 整数 | 6285 | WebUI 統計パネルの HTTP ポート |
| **統計パネルトークン** | 文字列 | 空 | パネルアクセスに必要な Token。空欄で認証なし |
| **会話遮断モード** | ドロップダウン | 空 | `ユーザー別` または `グローバル`。空欄で無効 |
| **会話遮断上限** | 整数 | 0 | この値に達すると API を遮断 |
| **会話遮断クールダウン** | 整数 | 0 | 上限到達後に復帰するまでの分数 |

### 設定例

**シナリオ1：API 残高の急速な消費を防止**
```
呼び出しクールダウン間隔：3秒
最大呼び出し回数：20回
クールダウン待機時間：10分
1日の呼び出しクオータ：100回
カスタム拒否メッセージ：リクエストが多すぎます。しばらく待ってください〜
```
> ボットは3秒ごとに最大1件返信し、20件連続送信後は10分間休止、1日最大100回。

**シナリオ2：深夜の迷惑を防止**
```
静穏時間開始：23:00
静穏時間終了：07:00
カスタム拒否メッセージ：現在は静穏時間です。明日また来てください〜
```
> 夜23時から朝7時までボットは API を完全に呼び出しません。

**シナリオ3：全面保護 + 管理者ホワイトリスト**
```
呼び出しクールダウン間隔：3秒
最大呼び出し回数：15回
クールダウン待機時間：5分
1日の呼び出しクオータ：200回
静穏時間開始：00:00
静穏時間終了：07:00
ホワイトリストユーザーID：123456789
```
> 管理者は一切制限されず、他のユーザーは昼間3秒ごとに1件返信、深夜は完全にブロック。

<summary><h2>📊 統計パネル</h2></summary>

**チャット統計**：`/webui` を送信して API レート制限の統計データを確認。

**WebUI パネル**：管理者は以下のコマンドで制御：

```bash
开启统计面板    # 起動、アクセスURL（Token付き）を返す
关闭统计面板    # パネルを停止
```

ブラウザで表示される内容：
- 本日の呼び出し回数 / 1日のクオータ
- 累計許可 / 累計ブロック / 許可率
- クールダウン状態プログレスバー
- 現在の全設定状態
- 最近のブロックログ

LAN アクセス + Token 認証に対応。未許可の閲覧を防止。

</details>
